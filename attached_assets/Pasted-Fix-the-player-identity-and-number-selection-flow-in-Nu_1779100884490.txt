Fix the player identity and number selection flow in Number Hunt.

Player nickname + serial system:
- When the player opens the app for the first time, show a first-time setup screen.
- The player must choose a nickname before entering the main app.
- After nickname submission, the system automatically generates a unique serial ID for that player.
- Display the player identity as:
  Nickname #Serial
  Example:
  Ahmed #48291
- Save nickname and serial locally using AsyncStorage.
- The player should not be asked for nickname again unless they reset profile from settings.
- Use this nickname + serial to track:
  - Solo records
  - Online matches
  - Wins/losses
  - Player statistics

Serial requirements:
- Serial is generated by the system only.
- Player cannot edit the serial.
- Serial should appear next to nickname immediately after nickname submission.
- Serial should be used in online rooms so players can identify each other.
- Example format:
  PlayerName #12345

Online multiplayer identity:
- Do not ask for nickname inside Online Multiplayer anymore.
- Use the saved nickname and serial from the first-time setup.
- Room screen should show:
  - My name: Nickname #Serial
  - Opponent name: Nickname #Serial
  - Room code
- If no nickname exists, redirect player to first-time setup before online mode.

Remove approve button for number entry:
- Remove the approve/confirm button after entering guesses.
- The guess should submit automatically once the player enters the full required number length.
- Example:
  - 2-digit mode: after entering 2 digits, submit guess automatically.
  - 3-digit mode: after entering 3 digits, submit guess automatically.
  - 4-digit mode: after entering 4 digits, submit guess automatically.
- After auto-submit:
  - Clear the input.
  - Add the guess to history.
  - Show feedback.
  - Allow next guess immediately if not correct.

Digit length selection:
- Remove any approve/confirm button after choosing digit length.
- When player taps:
  - 2 digits
  - 3 digits
  - 4 digits
- The selection is applied immediately.
- In online mode, host tapping digit length should start the game automatically for both players.

Bug prevention:
- Prevent duplicate auto-submission.
- Add a short lock/debounce after auto-submit.
- Do not submit if the input length is not complete.
- Do not crash if nickname, serial, room, digitLength, or socket is missing.
- Show friendly error messages instead of “Something went wrong”.

Testing checklist:
- First app launch asks for nickname.
- After nickname submission, serial appears next to nickname.
- App remembers nickname and serial after restart.
- Online mode uses saved nickname and serial.
- No nickname prompt appears inside online mode.
- Guess submits automatically after full number length.
- No approve button remains for guesses.
- Digit length selection starts immediately.
- No duplicate guesses are submitted.
